# Slack Mrkdwn Fixer

## 前提（重要）

Slack の MCP 下書き／送信ツールは、本文を **標準 Markdown** として解釈し、内部で
Slack mrkdwn へ変換してから表示する。実機検証で判明した変換:

| 入力（標準md） | 変換後（mrkdwn） | 表示 |
|------|------|-----------|
| `**x**` / `__x__` | `*x*` | 太字（※下記の境界条件が必要） |
| `*x*`（単一アスタリスク） | `_x_` | イタリック |
| `_x_`（単一アンダースコア） | `_x_` | イタリック（日本語隣接だと壊れる） |
| `~~x~~` | `~x~` | 取り消し線（※太字と同じ境界条件が必要） |
| `` `x` `` | `` `x` `` | コード（隣接文字の影響なし） |

### 境界条件（最重要・実機で確定）

`**x**` は変換後 `*x*`、`~~x~~` は `~x~` になる。Slack は `*x*` / `~x~` を
**開き記号の直前と閉じ記号の直後が「空白」か「行境界」のときだけ** 装飾描画する。
**全角文字（`・` `、` `を` `（` など）が記号に直接接触すると装飾されない（太字も
取り消し線も同じ）。**

検証結果（`・**作成数** —` 形式で開き記号の直前を変えてテスト）:

| 開き/閉じの外側 | 例 | 結果 |
|---|---|---|
| 半角スペース／行頭・行末 | `・ **作成数**` / `- **x**` / 行頭 `**x**` | ✅ 装飾される |
| 全角文字が密着 | `・**作成数**` / `そこで、**作成数**を` / `~~これ~~と` | ❌ 装飾されない |

→ **対策：強調スパンの外側が非空白なら半角スペースを挿入して境界を作る。**

## 何をするか

強調表記を **すべて太字 (`**bold**`) に統一** し、各スパンの外側に境界を確保する。

1. `*x*` → `**x**`（単一アスタリスクのイタリックを太字へ）
2. `_x_` → `**x**`（アンダースコアのイタリックを太字へ）
3. `__x__` → `**x**`（表記ゆれを `**` に統一）
4. `**x**` はそのまま（既に太字）
5. `~x~` → `~~x~~`（単一チルダを GFM 取り消し線へ統一）。`~~x~~` はそのまま。
   `` `code` `` もそのまま
6. 三連バッククォートのコードブロック・インラインコードの**中身**は変更しない
7. **各 `**...**` / `~~...~~` の外側が非空白文字なら半角スペースを挿入**
   （境界条件のため）。行頭・行末・既に空白の箇所には挿入しない。
   `・**x**` → `・ **x**`、`**x**を` → `**x** を`、`~~x~~と` → `~~x~~ と`。

## 使い方

Slack MCP に渡す本文をスクリプトで正規化し、その出力を `message` に渡す。

```bash
printf '%s' "$DRAFT" | python3 scripts/fix_slack_mrkdwn.py
```

複数行はヒアドキュメントで:

```bash
python3 scripts/fix_slack_mrkdwn.py <<'EOF'
・**作成数**を追う
これは_重要_な *ポイント* です
`code` はそのまま
EOF
# => ・ **作成数** を追う      ← ・と を に密着しないよう半角スペースを挿入
#    これは **重要** な **ポイント** です
#    `code` はそのまま
```

正規化後の本文を `slack_send_message_draft` / `slack_send_message` の `message` に渡す。

## 注意・限界

- 識別子のアンダースコア（`snake_case`・`user_id`・URL 内など、アンダースコア対の外側が両方とも ASCII 英数字）は強調と見なさず変更しない。
- スペースで囲まれた `*`（例: `2 * 3`）は変更されないが、スペースの無い `2*3*4` のような並びは強調として扱われ太字化され得る（チャット文では稀）。出力は目視確認すること。
- 平文中の `__dunder__`（前後が空白等）は太字化され得る。コード片は `` ` `` で囲めば保護される。
- 単一チルダ `~x~` は取り消し線 `~~x~~` に変換される。数値範囲 `10~20` のような単独チルダ（対になっていない）は変換されないが、`~20~` のように対になる並びは取り消し線化され得る。コード片は `` ` `` で囲めば保護される。
- 境界用の半角スペースは「外側が非空白なら一律挿入」する方針のため、ASCII 句読点（`(` `:` など）の前後でも挿入され `( **x** )` のようになり得る。日本語チャットでは問題ないが、気になる場合は出力を目視で微調整する。
- 検証は **下書きのプレビュー** で行うこと（送信前の見え方が確実）。`*x*` 形式は外側に空白が必要、という Slack 固有の挙動に依存している。

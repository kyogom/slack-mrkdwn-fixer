# Slack Mrkdwn Fixer

## 前提（重要）

Slack の MCP 下書き／送信ツールは、本文を **標準 Markdown** として解釈し、内部で
Slack mrkdwn へ変換してから表示する。実機検証で判明した変換:

| 入力（標準md） | 変換後（mrkdwn） | 表示 |
|------|------|-----------|
| `**x**` / `__x__` | `*x*` | 太字（※下記の境界条件が必要） |
| `*x*`（単一アスタリスク） | `_x_` | イタリック（※境界条件が必要） |
| `_x_`（単一アンダースコア） | `_x_` | イタリック（※境界条件が必要） |
| `~~x~~` | `~x~` | 取り消し線（※境界条件が必要） |
| `` `x` `` | `` `x` `` | コード（隣接文字の影響なし） |

Slack の基本装飾は太字・イタリック・取り消し線の3つで、下線は無い。

### 境界条件（最重要・実機で確定）

変換後の `*x*` / `_x_` / `~x~` を Slack が装飾描画するのは、
**開き記号の直前と閉じ記号の直後が「空白」か「行境界」のときだけ**。
**全角文字（`・` `、` `を` `（` など）が記号に直接接触すると装飾されない（太字・
イタリック・取り消し線のいずれも同じ）。**

検証結果（`・**作成数** —` 形式で開き記号の直前を変えてテスト）:

| 開き/閉じの外側 | 例 | 結果 |
|---|---|---|
| 半角スペース／行頭・行末 | `・ **作成数**` / `- **x**` / 行頭 `**x**` | ✅ 装飾される |
| 全角文字が密着 | `・**作成数**` / `そこで、**作成数**を` / `~~これ~~と` | ❌ 装飾されない |

→ **対策：強調スパンの外側が非空白なら半角スペースを挿入して境界を作る。**

## 何をするか

太字・イタリック・取り消し線をそれぞれ一つの記法に揃え、各スパンの外側に境界を確保する。
イタリックは太字に変換せず、イタリックのまま残す。

1. `**x**` はそのまま（既に太字）
2. `__x__` → `**x**`（太字の表記ゆれを `**` に統一）
3. `*x*` → `_x_`（イタリックを `_` に統一。太字にはしない）
4. `_x_` はそのまま（イタリックのまま残す）
5. `~x~` → `~~x~~`（単一チルダを GFM 取り消し線へ統一）。`~~x~~` はそのまま
6. 三連バッククォートのコードブロック・インラインコードの**中身**は変更しない
7. **各 `**...**` / `_..._` / `~~...~~` の外側が非空白文字なら半角スペースを挿入**
   （境界条件のため）。行頭・行末・既に空白の箇所には挿入しない。
   `・**x**` → `・ **x**`、`**x**を` → `**x** を`、`~~x~~と` → `~~x~~ と`。
   `_..._` は識別子（`snake_case` 等）を誤装飾しないよう、両端が識別子文字
   （英数字・アンダースコア）のときは触らない。

## 使い方

Slack MCP に渡す本文をスクリプトで正規化し、その出力を `message` に渡す。

```bash
printf '%s' "$DRAFT" | python3 scripts/fix_slack_mrkdwn.py
```

複数行はヒアドキュメントで:

```bash
python3 scripts/fix_slack_mrkdwn.py <<'EOF'
・**作成数**を追う
これは_強調_な *イタリック* です
`code` はそのまま
EOF
# => ・ **作成数** を追う          ← ・と を に密着しないよう半角スペースを挿入
#    これは _強調_ な _イタリック_ です   ← イタリックは _ のまま残す
#    `code` はそのまま
```

正規化後の本文を `slack_send_message_draft` / `slack_send_message` の `message` に渡す。

## 注意・限界

- 識別子のアンダースコア（`snake_case`・`user_id`・URL 内など、アンダースコア対の外側が両方とも識別子文字）はイタリックと見なさず変更しない。
- スペースで囲まれた `*`（例: `2 * 3`）は変更されないが、スペースの無い `2*3*4` のような並びは強調として扱われイタリック化され得る（チャット文では稀）。出力は目視確認すること。
- 平文中の `__dunder__`（前後が空白等）は太字化され得る。コード片は `` ` `` で囲めば保護される。
- 単一チルダ `~x~` は取り消し線 `~~x~~` に変換される。数値範囲 `10~20` のような単独チルダ（対になっていない）は変換されないが、`~20~` のように対になる並びは取り消し線化され得る。コード片は `` ` `` で囲めば保護される。
- 境界用の半角スペースは「外側が非空白なら一律挿入」する方針のため、ASCII 句読点（`(` `:` など）の前後でも挿入され `( **x** )` のようになり得る。日本語チャットでは問題ないが、気になる場合は出力を目視で微調整する。
- 検証は **下書きのプレビュー** で行うこと（送信前の見え方が確実）。装飾は外側に空白が必要、という Slack 固有の挙動に依存している。
